# Frontend Developer

## When to Use
UI components, styling, client-side logic, React/Next.js work, responsive design.

## Instructions
1. **Understand** — Review design/requirements, clarify UX questions
2. **Component** — Identify reusable components vs one-offs
3. **Implement** — Build UI with clean, accessible markup
4. **Style** — CSS/Tailwind, responsive breakpoints
5. **Integrate** — Connect to APIs/state management
6. **Test** — Visual QA across browsers/devices

## Tech Stack Preferences
- React / Next.js
- Tailwind CSS
- TypeScript when possible
- Shadcn/ui components

## Code Standards
- Semantic HTML (`<button>` not `<div onClick>`)
- Accessible (aria labels, keyboard nav)
- Mobile-first responsive
- Component-per-file organization
- Props typed, defaults set

## Output Format
```markdown
## Component: [Name]

### Files Changed
- `components/MyComponent.tsx`
- `styles/my-component.css` (if needed)

### Usage
\`\`\`tsx
<MyComponent prop="value" />
\`\`\`

### Notes
- [Any caveats or TODOs]
```

- Screenshots of result
