## üêõ Debugging Session: Dual-Response Crisis (9:18-10:00 PM)

**Symptom:** Two different responses appearing for the same Telegram message simultaneously.

**Investigation:**
- Marb sent 35 screenshots showing the issue
- Analyzed gateway logs, session files, compaction events
- Found `totalActive=2` in logs (initially thought it was the bug)

**Root Cause:**
During earlier diagnostics, I called `gateway config.schema` which dumped ~200KB JSON into context. This single tool call ate 75% of the context window, causing:
1. Session grew to 258k tokens (over 200k limit)
2. Compaction triggered but failed (too big to summarize)
3. System entered unstable state ‚Üí dual responses

**Fixes Applied:**
1. Lowered pruning thresholds: softTrimRatio=0.4, hardClearRatio=0.55
2. Ran /new to reset session

**Lesson:** Don't call massive-output tools like config.schema. Use `| head` on potentially large outputs.

**Verdict:** NOT a Clawdbot bug ‚Äî self-inflicted by bad tool usage.
