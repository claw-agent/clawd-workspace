### Agent Limitations (Still!)
- Makes subtle conceptual errors (like a hasty junior dev)
- Wrong assumptions without checking
- Doesn't seek clarifications or surface inconsistencies
- Too sycophantic, doesn't push back
- Overcomplicates code, bloats abstractions
- Doesn't clean up dead code
- Will write 1000 lines when 100 would do

### What Works
- **Tenacity:** Agents never get tired, keep trying where humans give up
- **Leverage:** Give success criteria, not instructions. Let it loop.
- **Tests first:** Write tests, then have agent pass them
- **Browser MCP:** Put agent in loop with browser
