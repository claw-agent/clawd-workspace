### The Core Argument

Will introduces the concept of a **"Tool Shaped Object" (TSO)** â€” something that looks like a tool, feels like a tool, produces the *sensation* of work... but doesn't actually produce work. **Its function IS to feel like a tool.**

**The Kanna Analogy:** A $3,000 Japanese hand plane that takes days to set up and produces transcendent wood shavings â€” but a power planer does the same work in seconds. The setup IS the practice. The shavings ARE the product.

### Key Arguments

1. **"The market for feeling productive is orders of magnitude larger than the market for being productive."** â€” This is the essay's most-quoted line.

2. **FarmVille as paradigm**: Click anywhere, number goes up. The direction doesn't matter. The screen fills with evidence of effort. This is the entire product.

3. **The AI boom is the most sophisticated TSO ever created:**
   - We talk about token budgets, GPU clusters, trillion-dollar buildouts
   - "AI is everywhere in consumption and almost nowhere in output"
   - Teams build agent systems of "breathtaking complexity whose primary output is the existence of the system itself"
   - Agents produce logs â†’ logs analyzed by other agents â†’ reports generated â†’ dashboards populated â†’ the apparatus hums with "the unmistakable energy of work being done" â†’ **what is being done is the operation of the apparatus**

4. **The token budget illusion**: Miles Grimshaw's "token budget" concept treats token consumption like capex â€” more tokens = more output. Will argues the relationship between tokens consumed and value produced is "in most cases, a cloud" not a line.

5. **Why LLMs are uniquely dangerous as TSOs**: Prior TSOs were constrained by medium (FarmVille â†’ sensation of farming, Notion â†’ sensation of organizing). **An LLM can produce the sensation of anything.** 

6. **The gradient problem**: LLMs ARE genuine tools sometimes. The line between tool and TSO isn't a line but a gradient. "You can only fail to notice when you have crossed from one side to the other."

7. **The prescription**: "Ask what the number is before making it go up."

### How This Applies to AI Tool Evaluation

This is a **critical framework** for evaluating any AI tool or workflow:

| Question | TSO Indicator |
|----------|--------------|
| What measurable output does this produce? | If you can't name it â†’ TSO |
| Could I achieve the same result without this tool? | If yes with less effort â†’ TSO |
| Am I configuring/tuning more than producing? | Setup-as-practice â†’ TSO |
| Does using this tool make me FEEL productive? | Feeling â‰  being â†’ check |
| What would happen if I stopped using this? | If nothing changes â†’ TSO |
